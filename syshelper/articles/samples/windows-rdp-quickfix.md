# Windows RDP: быстрый чеклист, если не пускает

## Базовый сценарий

Когда RDP не подключается, обычно надо проверить:

1. включён ли удалённый доступ;
2. слушает ли порт;
3. открыт ли firewall;
4. нет ли ошибки в NAT / VPN / маршруте.

## Проверка порта

```powershell
netstat -an | find "3389"
```

Если слушаешь нестандартный порт:

```powershell
netstat -an | find "3390"
```

## Firewall

```powershell
Get-NetFirewallRule -DisplayGroup "Remote Desktop"
```

## Проверка пользователя

- пользователь должен иметь право на вход по RDP;
- на Home редакциях обычный RDP host не работает;
- иногда мешает NLA.

> [!WARNING]
> Не отключай безопасность просто ради теста надолго. Делай временную диагностику, а потом возвращай защищённую конфигурацию.

## Быстрый итог

- локально сервис работает;
- по IP внутри сети заходит;
- через NAT уже не заходит;
- значит проблема обычно не в Windows, а в маршруте, firewall или пробросе.

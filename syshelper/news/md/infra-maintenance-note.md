# Плановая заметка: как вести сервисные апдейты

## Рекомендуемый формат

1. что изменили;
2. когда изменили;
3. где изменили;
4. были ли риски;
5. что проверить после изменений.

> [!WARNING]
> Не пиши новости в формате «что-то поправили». Через неделю это уже бесполезно. Нужен короткий, но понятный контекст.

## Хороший шаблон

```md
# Название обновления

**Дата:** 2026-04-21  
**Зона:** Router / Server / Linux workstation  

## Что сделали
- ...

## Что проверить
- ...

## Результат
- ...
```
